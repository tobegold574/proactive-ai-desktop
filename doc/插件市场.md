# 插件市场 — 工程方案

本文约定：插件体系分 **三期** 交付，每期有明确代码边界与发布标准。**一期全部为离线本地插件**；二期增加签名与远程索引；三期增加许可证与远端执行能力（可选模块）。未写入本期范围的特性，实现时不展开平行设计，避免过度抽象。

---

## 1. 现状与衔接

- **现状（代码）**：主进程已有 **`PluginRegistry`**、`plugin-preferences`（electron-store）、`chat:send` 前后钩子链（含 `onSystemPromptBuild` / `onMessageReceive` 等）、IPC **`plugins:list`** / **`plugins:setEnabled`**；设置页有插件列表与开关。内置插件目录 **`src/main/plugins/builtin/`** 当前 **仅注册 `com.proactiveai.pavatar`**（其它文档中曾写的导出 / 片段 / 记忆通知等 **未作为内置挂载**；`exportConversationMarkdown` 仅在 registry 内保留方法，**无 IPC 暴露**）。
- **本期（一期）目标（产品）**：继续以 **离线、安全默认** 为原则扩展；像素小人资源包协议见 **`doc/像素小人插件.md`**。**无远程市场 UI**。
- **差异化**：像素小人已按 **独立 pack + 内置 hook 插件** 落地，与二期 `.paplugin` 动态加载可再对齐。

---

## 2. 三期总览

| 期 | 范围 | 发布标准 |
| --- | --- | --- |
| **一期** | 内置官方插件 + 本地 `plugins/` 目录加载；manifest `schema_version: 1`；权限模型为布尔开关 | 卸载重装后插件可重现；钩子异常不拖垮发送流程 |
| **二期** | 远程 `index.json`（HTTPS）列举插件；`.paplugin` 签名包安装/升级；渲染进程「插件」设置页 | 公钥轮换策略文档化；失败回滚上一版本 |
| **三期** | 许可证在线校验（可选离线宽限）；**远端技能**单独进程与单独权限，与本地插件包解耦 | 安全评审通过；默认关闭远端执行 |

## 3. 一期：目录结构、manifest、加载模型

### 3.1 用户数据目录

在 `app.getPath('userData')` 下：

```text
userData/
├── plugin-preferences.json    # 启用状态与配置（推荐 electron-store，见 §5）
└── plugins/
    ├── _builtin/                # 随应用更新，只读镜像（也可直接打进 asar，由 Loader 优先读 asar）
    │   ├── export-markdown/
    │   ├── snippet-transform/
    │   └── memory-notify/
    └── custom/
        └── <plugin_id>@<semver>/
            ├── manifest.json
            ├── main.cjs         # 单入口，CommonJS，由主进程 require
            └── README.md        # 可选
```

**硬性规则**：

- 主进程 **仅** `require(main.cjs)` 于 `plugins/custom/*` 与内置目录；**禁止** `require` 用户任意路径。
- `main.cjs` 必须 **同步导出** `function createPlugin(context): PluginRuntime`，见 §3.4。

### 3.2 `manifest.json`（一期）

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `schema_version` | number | 是 | 一期固定 `1` |
| `id` | string | 是 | 反向域名，如 `com.proactiveai.export-markdown` |
| `version` | string | 是 | SemVer |
| `name` | string | 是 | 展示名 |
| `main` | string | 是 | 相对路径，一期固定 `main.cjs` |
| `min_app_version` | string | 是 | 低于则拒绝加载 |
| `permissions` | string[] | 是 | 见 §3.3 |
| `hooks` | string[] | 是 | 声明启用的钩子名，必须是 `PluginHooks` 键的子集 |

### 3.3 权限枚举（一期）

实现为 **字符串字面量联合**，未在 manifest 声明则对应 API 在 `context` 中不存在或调用抛错。

| permission | 含义 | 授予的 context API |
| --- | --- | --- |
| `messages.read` | 读取当前会话消息列表 | `getMessages(conversationId)` |
| `fs.writesDownloads` | 仅允许写入下载目录 | `writeToDownloads(filename, content)` |
| `clipboard.write` | 写剪贴板 | `setClipboardText(text)` |
| `config.read` | 读 GlobalSettings 脱敏副本 | `getPublicSettings()`（不含 apiKey） |

**一期不授予**：任意路径读写、子进程、`net` 原始请求。

### 3.4 `PluginRuntime` 与上下文

**`createPlugin(context)` 返回**：

```ts
interface PluginRuntime {
  hooks: Partial<PluginHooks>
  dispose?: () => void | Promise<void>
}
```

**执行顺序（硬性）**：

- `onMessageSend`：**按插件 id 字典序**依次调用；前一个返回值作为下一个的输入。
- `onMessageReceive`：**逆序**调用（后装先生效），便于后处理。
- `onTrigger` / `onMemoryUpdate`：**字典序**；单个钩子抛错 **记录日志并跳过**，不影响其它插件与同次请求。

**超时（硬性）**：每个钩子 **500ms**（可配置常量）；超时则中止该钩子并标记 `degraded`。

### 3.5 挂载点（与现有代码对齐）

| 挂载点 | 文件 | 调用时机 |
| --- | --- | --- |
| Send 前 | `main/index.ts` 内 `chat:send`，在构造 `userMessage` 落盘前对用户输入文本调用 `onMessageSend` | 使用**即将发送**的字符串 |
| Send 后 | `chat:send`，收到 `AIResponse` 后、写入 assistant 消息前对 `reply` 调用 `onMessageReceive` | |
| 记忆合并后 | `chat:send` 内 `memory` 更新成功后调用 `onMemoryUpdate` | 传入**本次合并后的** `important_info` 数组 |
| 触发点 | `renderer` 侧 `useProactiveChat` 在 `addMessage` 前通知主进程 → 或由主进程统一调度（一期选 **主进程 IPC `plugins:emitTrigger`** 由渲染进程转发，避免双实现） | **一期简化**：仅在主进程 `chat:send` 返回前对 `triggers` 逐个调用 `onTrigger`（不等待定时器触发），用于日志类插件；**定时触发后的 UI 消息**在二期再统一到主进程调度 |

> **一期取舍说明**：定时触发的 assistant 消息仍在渲染进程产生；若官方插件需要「导出含 proactive 消息」，在二期把定时触发改为主进程 `messageStore.add` 或与一期增加 **renderer→main `messages:proactive:add`** 单一路径。本文 **一期** 采用：**onTrigger 仅同步作用于 API 返回的 trigger 列表**，不拦截稍后 UI 追加的文本。

## 4. 当前内置插件（与代码一致）

### 4.1 `com.proactiveai.pavatar`（2D 虚拟形象）

- **实现**：`src/main/plugins/builtin/pavatar.ts`
- **permissions**：`ui.dispatch`（向渲染进程派发 `AVATAR_SET_MOOD` / `AVATAR_PLAY_EMOTE`）
- **hooks**：`onSystemPromptBuild`（注入 `[[AVATAR:…]]` 协议说明）、`onMessageReceive`（解析标签、改写 reply、dispatch 表情）
- **资源**：`userData/pavatar-packs/` + 内置演示包同步；详见 **`doc/像素小人插件.md`**

### 4.2 规划中 / 曾文档化但未挂载的内置能力

以下条目 **仍可作为产品 backlog**；实现时需重新加入 `BUILTINS` 并补 IPC / UI：

- **导出会话 Markdown**（原 `com.proactiveai.export-markdown`）：registry 内可有实现，但 **preload 未暴露 `plugins:exportConversation`** 时设置页无法调用。
- **片段替换**、**记忆通知** 等同理。

## 5. `plugin-preferences.json`（启用与配置）

**推荐**：使用 `electron-store` 新文件 `plugin-preferences.json`：

```json
{
  "enabled": ["com.proactiveai.pavatar"],
  "config": {
    "com.proactiveai.pavatar": {
      "activePackId": "com.proactiveai.demo",
      "activePackVersion": "1.0.0"
    }
  }
}
```

**默认**：当前默认启用 **`com.proactiveai.pavatar`**；老配置若缺该 id，store 启动时会 **补全一次**（见 `plugin-preferences-store.ts`）。用户关闭插件后从 `enabled` 移除对应 id。

## 6. 二期：远程市场索引与 `.paplugin` 包

### 6.1 索引 `index.json`（HTTPS 固定 URL，配置在 `GlobalSettings.marketplaceIndexUrl`）

```json
{
  "schema_version": 1,
  "updated_at": 1713000000000,
  "plugins": [
    {
      "id": "com.example.foo",
      "latest": "1.2.0",
      "download_url": "https://cdn.example.com/plugins/foo-1.2.0.paplugin",
      "sha256": "...",
      "signature": "...",
      "min_app_version": "1.1.0"
    }
  ]
}
```

### 6.2 `.paplugin` 格式

- 实质：**ZIP**，内含与 §3.1 `custom` 目录相同结构。
- **验签**：与 `doc/社区对接.md` 中 `CommunityBundle` 相同 **Ed25519** 策略（可共用 `verifySignedPayload` 工具函数）。

### 6.3 安装流程

```
拉取 index.json → 校验 schema_version → 用户点安装
    → 下载 .paplugin → sha256 → 验签 → 解压到 plugins/custom/<id>@<version>/
    → 校验 min_app_version → Loader.reload()
    → 失败：删除临时目录，保留旧版本
```

## 7. 三期：许可证与远端技能（概要，实现前需单独评审）

| 组件 | 说明 |
| --- | --- |
| 许可证 | 启动时 `POST /license/verify` 携带 `plugin_id` + `device_id`（匿名 UUID）+ `user_sub`（若已登录）；返回 `valid_until`；本地缓存 7 天宽限 |
| 远端技能 | **独立** `PluginHost` 子进程，IPC 仅允许 `invokeSkill(name, payload)`；**禁止**与主进程共享 `require`；默认关闭 |

三期未立项前 **不得** 在一期代码中引入子进程加载用户脚本。

## 8. IPC 规划

| 通道 | 方向 | 说明 |
| --- | --- | --- |
| `plugins:list` | renderer→main | 返回 `PluginListEntry[]`（`id, name, version, enabled, builtin, error?`） |
| `plugins:setEnabled` | renderer→main | `pluginId, enabled`；成功后 main→renderer **`plugins:preferencesChanged`** |
| `plugins:setConfig` | renderer→main | （规划中）`id, config: object` |
| `plugins:exportConversation` | renderer→main | **未接 IPC**；若产品需要导出，需在 `main/index.ts` 注册并与 preload 对齐 |
| `plugins:reload` | main内部 | （规划中）热重载 |

二期增加：`plugins:installFromUrl`, `plugins:uninstall`。

## 9. 安全清单（一期发布前必查）

- [ ] `main.cjs` 加载前读取 manifest 与 `permissions`，**context 按权限裁剪**。
- [ ] `require` 路径限制在插件目录内；**禁止**动态 `require(userPath)`。
- [ ] 钩子 try/catch + 超时；错误仅记 `plugin_id` 与 stack 摘要。
- [ ] `getPublicSettings()` **过滤** `apiKey`、任何 token。

## 11. 与类型定义的映射

`PluginHooks` 与 `Plugin`（`shared/types.ts`）保持源类型；运行时以各内置模块返回的 **`Partial<PluginHooks>`** 为实际调用对象。二期动态加载 manifest 与 `hook_mismatch` 校验按本文 §3.2 再实现。

`UX与数据协议.md`「实现进度」：**插件 IPC 与像素小人已接**；内置插件数量以 **`BUILTINS` 数组** 为准（当前为 pavatar 一个）；远程市场为 S5。
