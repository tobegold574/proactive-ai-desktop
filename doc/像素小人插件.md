# 像素小人（2D Avatar）— 方案与当前实现

本文描述 **ProactiveAI Desktop** 中的 2D 虚拟形象能力：资源包协议、主进程与渲染层分工、以及与内置插件 `com.proactiveai.pavatar` 的集成。

> **实现状态（与仓库代码一致）**：资源包扫描、`pavatar://` 自定义协议、内置演示包同步、设置页插件开关、右侧形象栏（可折叠）、Web Worker 内轻量状态机 + 画布渲染、模型回复末尾 `[[AVATAR:…]]` 协议。**未实现**：`.pavatar` ZIP 安装/卸载 UI、斜杠指令 `/avatar …`、独立「皮肤库」页面、PixiJS（当前非 Pixi）。

---

## 1. 非目标（产品边界）

- 不做 3D（VRM/Live2D 等）。
- 不做网络同步「云皮肤」、第三方广告 SDK。
- 不做摄像头/语音驱动全身动捕。
- 不做第三方插件注入任意 React 组件（先做 **manifest + 静态资源**）。

---

## 2. 免费素材与许可（作者参考）

自制 pack 时建议优先 **CC0** 或可明确商用的许可；包内宜保留 `LICENSE` / 署名信息（与是否强制校验无关，便于分发）。内置演示包见 `resources/pavatar-bundled/…`。

## 3. 用户体验（已实现）

- **位置**：主界面 **右侧** 形象栏（可折叠），非右下角悬浮（旧方案已废弃）。
- **设置**：「插件」列表中可 **启用/禁用** `com.proactiveai.pavatar`；禁用后钩子与 dispatch 不生效。
- **缩放**：形象栏内 **显示倍率** 滑块（`displayScale`）；调节的是绘制倍率，不改变 pack 内像素定义。
- **表情历史**：展示近期切换的表情 id（由 `plugin:dispatch` 采集）。
- **与模型的约定**：插件通过 `onSystemPromptBuild` 注入说明，要求模型在 **reply 最后一行** 输出唯一一行  
  `[[AVATAR:<expression_id>]]`。主进程 `onMessageReceive` 解析后 **去掉该行** 再落盘/展示，并向渲染进程 **dispatch** `AVATAR_SET_MOOD`（及可选时长）。
- **聊天区**：消息列表 **不显示** 小头像图标；输入区与消息区水平 padding 为 `px-4 md:px-6 lg:px-8`。

> **尺度提示**：若 `idle` 雪碧 **单帧宽高** 与 `atlas` **单格 tileW/tileH** 不一致，同一 `displayScale` 下 idle 与表情在屏幕上的大小会 **跳变**。自制 pack 时建议让 idle 帧与 atlas 格子 **像素尺寸一致**，或后续在 manifest 中扩展统一倍率（当前未实现）。

## 4. 资源包目录与 `manifest.json`（v1）

### 4.1 磁盘布局

扫描根目录：

```text
<userData>/pavatar-packs/<packId>/<version>/manifest.json
                              └── idle.sheet.png, atlas.png, …
```

开发环境下，应用启动时会将仓库内 **内置演示包** 同步到上述目录（仅覆盖该 id+version，其它用户目录不动）：

- **源码侧**：`resources/pavatar-bundled/com.proactiveai.demo/1.0.0/`
- **打包侧**：`extraResources` → `process.resourcesPath/pavatar-bundled/…`

主进程从 `out/main` 起算，内置源目录为 **`../../resources/pavatar-bundled`**（勿写成多一层 `../`）。

### 4.2 `manifest.json` 字段

与 `src/shared/types.ts` 中 `PAvatarPackManifestV1` 一致，核心字段：

| 字段 | 说明 |
| --- | --- |
| `v` | 固定 `1` |
| `packId` / `version` / `name` | 包标识与展示名 |
| `author` / `license` | 可选 |
| `idle` | `kind: "sheet"`，`src`（相对包目录）、`frameW` / `frameH`、`frames`、`fps` |
| `atlas` | `src`、`cols` / `rows`、`tileW` / `tileH` |
| `expressions` | 可选；`表情 id → { row, col }`（0-based 网格）；省略时 Worker 使用内置默认映射 |

示例见 `resources/pavatar-bundled/com.proactiveai.demo/1.0.0/manifest.json`。

### 4.3 自定义协议 `pavatar://`

主进程 `registerFileProtocol('pavatar', …)`：URL 形态为 **`pavatar://<packId>/<version>/<相对路径>`**，用于把包内 `idle` / `atlas` 图片以 URL 形式交给渲染进程加载（`PAvatarPackResolved.idleUrl` / `atlasUrl`）。

## 5. 技术架构

- **主进程**：`src/main/pavatar/pack-store.ts` — 同步内置包、`scanPAvatarPacks`、`getActivePAvatarPackResolved`、协议注册；活跃包 id/version 存在 **`plugin-preferences` 里该插件的 `config`**（`activePackId` / `activePackVersion`）。
- **内置插件**：`src/main/plugins/builtin/pavatar.ts` — `onSystemPromptBuild`、`onMessageReceive`、向 `plugin:dispatch` 发 `AVATAR_SET_MOOD` / `AVATAR_PLAY_EMOTE`。
- **渲染进程**：`src/renderer/components/AvatarWidget/` — React UI；`src/renderer/avatar/engine.worker.ts` — **OffscreenCanvas / 位图渲染**（nearest-neighbor 风格缩放）、`postMessage` 与 UI 通信。

### 5.1 IPC / 事件（Electron）

| 名称 | 方向 | 说明 |
| --- | --- | --- |
| `pavatar:listPacks` | renderer→main | 已扫描到的 `PAvatarPackResolved[]` |
| `pavatar:getActivePack` | renderer→main | 当前活跃包或 `null` |
| `pavatar:setActivePack` | renderer→main | `packId, version`；成功则 `webContents.send('pavatar:activePackChanged', …)` |
| `plugins:list` / `plugins:setEnabled` | renderer→main | 见 `doc/插件市场.md` |
| `plugins:preferencesChanged` | main→renderer | 插件启用状态变更 |
| `plugin:dispatch` | main→renderer | `PluginDispatchMessage`（pavatar 情绪/表情） |

Preload：`electronAPI.pavatar.*`、`electronAPI.plugins.*`（含 `onPreferencesChanged`）。

## 8. 排错摘要

- **`pavatar:getActivePack` 等报错未注册**：主进程未走到最新 `setupIPC` 或进程未重启；开发时请 **完全退出 Electron 再 `npm run dev`**。
- **`syncBundledPavatarPack: missing bundled source dir`**：内置资源路径错误；确认 `resources/pavatar-bundled` 存在且主进程 `__dirname` 层级与 `pack-store.ts` 一致。
- **设置里插件列表为空**：多为 **旧 preload** 无 `plugins.list`；全退应用后重开；界面若提示 bridge 不可用，同因。
